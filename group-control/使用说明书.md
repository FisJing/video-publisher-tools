## 1. 系统简介

微信视频号群控系统是一个基于 ADB + WebSocket 的自动化发布工具，支持同时管理多台手机，批量发布视频到微信视频号。

### 系统架构

```
┌─────────────────┐     WebSocket      ┌─────────────────┐
│   Web管理界面    │◄──────────────────►│    服务器       │
│ (localhost:3000)│                    │  (Node.js)      │
└─────────────────┘                    └────────┬────────┘
                                                │
                                                │ ADB
                                                ▼
                                    ┌───────────────────────┐
                                    │   手机设备 (多台)      │
                                    │  - 资产-13            │
                                    │  - 资产-5             │
                                    │  - ...                │
                                    └───────────────────────┘
```

## 4. 安装部署

### 方式一：一键安装（推荐）

1. 确保已安装 Node.js
2. 双击运行 `install.bat`
3. 按提示完成安装

### 方式二：手动安装

#### 4.1 安装 Node.js

1. 访问 https://nodejs.org
2. 下载 LTS 版本
3. 安装时勾选 "Add to PATH"

#### 4.2 安装 Scrcpy + ADB

1. 下载 Scrcpy: https://github.com/Genymobile/scrcpy/releases
2. 解压到任意目录（如 `C:\Scrcpy`）
3. 确保目录下有 `adb.exe`

#### 4.3 安装项目依赖

```bash
cd group-control
npm install
```

#### 4.4 配置ADB路径

编辑 `server.js` 第15行：

```javascript
const ADB_PATH = 'C:\\Users\\你的用户名\\Desktop\\Scrcpy\\adb.exe';
```

#### 4.5 设置开机自启

将 `start.bat` 的快捷方式放入启动文件夹：

```
Win + R → 输入 shell:startup → 粘贴快捷方式
```

## 5. 快速开始

### 5.1 启动服务器

**方式一：双击运行**
```
双击 start.bat
```

**方式二：命令行运行**
```bash
cd group-control
node server.js
```

**方式三：开机自动启动**
```
已配置开机自启，登录Windows后自动运行
```

### 5.2 访问管理界面

打开浏览器访问：

```
http://localhost:3000
```

### 5.3 连接手机设备

#### 有线连接

1. USB连接手机到电脑
2. 手机授权USB调试
3. 刷新页面查看设备

#### 无线连接（推荐）

1. 手机和电脑连接同一WiFi
2. 获取手机IP地址：设置 → WLAN → 点击网络
3. USB连接手机后运行：
   ```bash
   adb tcpip 5555
   adb connect 手机IP:5555
   ```
4. 拔掉USB线
5. 配置 `device-config.json`：
   ```json
   {
     "手机IP:5555": { "alias": "设备名称", "description": "视频号" }
   }
   ```
6. 重启服务器自动连接

### 5.4 发布视频

1. 准备Excel文件（格式见下方）
2. 点击"扫描Excel文件"或手动输入路径
3. 选择设备
4. 点击"开始发布"

#### Excel格式要求

| 视频标签 | 视频位置 |
|----------|----------|
| 描述文字1 | C:\videos\video1.mp4 |
| 描述文字2 | C:\videos\video2.mp4 |

## 6. 功能说明

### 6.1 设备管理

#### 设备列表

- 显示所有已连接设备
- 设备类型：ADB / WebSocket
- 设备状态：在线/离线
- 设备别名和描述

#### 设备配置

点击设备卡片可编辑：
- **别名**：自定义设备名称
- **描述**：设备备注信息

### 6.2 任务管理

#### 创建任务

1. 选择设备
2. 输入视频路径
3. 输入视频描述
4. 选择微信类型（机主/分身）
5. 点击"创建任务"

#### 任务队列

- 多任务自动排队
- 显示队列位置
- 支持停止任务

#### 任务状态

| 状态 | 说明 |
|------|------|
| pending | 等待执行 |
| running | 正在执行 |
| completed | 执行完成 |
| failed | 执行失败 |
| stopped | 已停止 |

### 6.3 发布流程

系统自动执行以下步骤：

```
步骤0:  清空视频      - 清空手机视频目录
步骤1:  推送视频      - 将视频传到手机
步骤2:  重启微信      - 强制停止并重启
步骤3:  点击发现      - 进入发现页
步骤4:  点击视频号    - 进入视频号
步骤5:  点击个人      - 进入个人主页
步骤6:  点击发表      - 点击发表按钮
步骤7:  点击相册      - 选择相册
步骤8:  点击图片视频  - 切换标签
步骤9:  点击所有视频  - 查看所有
步骤10: 选择视频      - 选择视频
步骤11: 点击选择      - 确认选择
步骤12: 点击完成1     - 第一次完成
步骤13: 点击完成2     - 第二次完成
步骤14: 点击描述框    - 激活输入
步骤15: 输入描述      - 输入文字
步骤16: 点击发表      - 发布视频
```

### 6.4 分身微信

系统支持分身微信（如应用分身、多开）：

| 用户ID | 类型 | 说明 |
|--------|------|------|
| 0 | 机主微信 | 默认微信 |
| 999 | 分身微信 | 应用分身 |

在设备配置或任务创建时选择微信类型。

## 7. 设备配置

### 7.1 配置文件

设备配置保存在 `device-config.json`：

```json
{
  "192.168.31.163:5555": {
    "alias": "资产-13",
    "description": "视频号"
  },
  "192.168.31.167:5555": {
    "alias": "资产-5",
    "description": "视频号"
  }
}
```

### 7.2 添加新设备

1. 获取手机IP地址
2. 开启无线ADB连接
3. 编辑 `device-config.json` 添加配置
4. 重启服务器

### 7.3 坐标配置

如果手机分辨率不同，需要调整点击坐标。

编辑 `server.js` 或创建任务时指定坐标：

```javascript
const POS = {
    discover: [675, 2300],      // 发现
    videoChannel: [326, 500],   // 视频号
    personal: [1000, 120],      // 个人
    publishBtn: [240, 1850],    // 发表
    album: [100, 2300],         // 相册
    tagLabel: [540, 200],       // 图片视频
    allVideos: [540, 400],      // 所有视频
    videoItem: [200, 400],      // 视频项
    selectBtn: [972, 150],      // 选择
    completeBtn1: [980, 2300],  // 完成1
    completeBtn2: [940, 2250],  // 完成2
    descInput: [540, 1072],     // 描述框
    finalPublish: [1000, 180]   // 发表
};
```

#### 获取坐标方法

```bash
# 手机上点击位置后
adb shell input tap x y
```

或使用开发者选项中的"指针位置"功能。

## 8. 常见问题

### Q1: 设备连接失败？

**原因：**
- 手机和电脑不在同一WiFi
- 手机IP地址变化
- 手机未开启ADB调试

**解决：**
1. 检查WiFi连接
2. 重新获取手机IP
3. 更新 `device-config.json`
4. 重启服务器

### Q2: 任务执行失败？

**原因：**
- 视频文件不存在
- 微信未登录
- 坐标位置错误

**解决：**
1. 检查视频路径是否正确
2. 确认微信已登录
3. 调整点击坐标

### Q3: 文字输入失败？

**原因：**
- 未安装ADBKeyboard
- 输入法切换失败

**解决：**
1. 安装 ADBKeyboard APK
2. 手动切换输入法测试
3. 检查 `server.js` 中的输入命令

### Q4: 服务器启动失败？

**原因：**
- Node.js未安装
- 端口被占用
- 依赖未安装

**解决：**
1. 运行 `node -v` 检查Node.js
2. 修改 `server.js` 中的端口
3. 运行 `npm install`

### Q5: 开机不自启？

**原因：**
- 启动快捷方式丢失

**解决：**
1. 按 `Win + R`，输入 `shell:startup`
2. 检查是否有"微信视频号群控"快捷方式
3. 重新运行 `install.bat`

### Q6: 看不到设备？

**原因：**
- ADB未连接
- 设备离线

**解决：**
```bash
# 检查ADB连接
adb devices

# 手动连接
adb connect 手机IP:5555
```
