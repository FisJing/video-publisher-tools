# 微信视频号群控系统

基于 ADB + WebSocket 的微信视频号自动发布系统，支持多设备、多账号、Web管理界面。

## ✨ 功能特性

- 🖥️ **Web管理界面** - 可视化操作，无需命令行
- 📱 **多设备管理** - 支持多台手机同时控制
- 🔄 **实时投屏** - 在网页上实时查看手机屏幕
- 📝 **多语言输入** - 支持中文、俄文等所有语言
- 🎯 **任务队列** - 支持批量任务自动排队执行
- 📊 **Excel导入** - 从Excel文件读取视频列表
- 🔧 **分身微信支持** - 完美支持微信分身发布

## 🚀 快速开始

### 1. 环境要求

- **Node.js** 14+ 
- **ADB** (Android Debug Bridge)
- 手机开启 **USB调试** 和 **无线调试**

### 2. 安装

```bash
# 安装依赖
npm install

# 或使用快捷脚本
install.bat
```

### 3. 启动服务

```bash
node server.js

# 或双击
start.bat
```

服务启动后打开浏览器访问：**http://localhost:3000**

### 4. 连接设备

**方法一：USB连接**
```bash
adb devices
adb tcpip 5555
adb connect 手机IP:5555
```

**方法二：无线连接（Android 11+）**
1. 手机设置 → 开发者选项 → 无线调试
2. 点击"使用配对码配对设备"
3. 记下配对端口（如：192.168.31.163:37891）
4. 运行：
```bash
adb pair 192.168.31.163:37891
# 输入配对码
adb connect 192.168.31.163:5555
```

### 5. 安装输入法

在Web界面点击"安装ADBKeyboard"按钮，或手动安装：
```bash
adb install ADBKeyboard.apk
```

## 🎮 使用说明

### Web界面操作

1. **设备管理** - 自动检测并列出所有连接的设备
2. **选择视频** - 点击"选择Excel文件"导入视频列表
3. **创建任务** - 选择设备和微信类型（机主/分身）
4. **实时监控** - 通过投屏功能查看发布进度

### Excel格式

| 视频标签 | 视频位置 |
|---------|---------|
| 视频描述1 | C:\Videos\video1.mp4 |
| 视频描述2 | C:\Videos\video2.mp4 |

### 分身微信设置

系统已自动处理分身微信的输入法问题：
- 自动禁用分身空间的系统输入法
- 自动设置ADBKeyboard为默认输入法
- 使用Base64编码确保多语言输入正确

## ⚙️ 配置说明

### 设备配置 (device-config.json)

```json
{
  "192.168.31.163:5555": {
    "alias": "设备别名",
    "description": "设备描述"
  }
}
```

### 点击坐标配置

坐标基于 **1080x2412** 分辨率，如需修改请编辑 `server.js` 中的坐标值。

## 🔧 常见问题

### Q: 分身微信无法输入中文？

**A:** 系统已自动解决此问题：
1. 自动禁用分身空间的百度/搜狗输入法
2. 自动设置ADBKeyboard为默认
3. 使用Base64编码传输文本

### Q: 截图失败/投屏卡顿？

**A:** 可能原因：
- WiFi信号弱，建议使用5G频段
- 手机性能不足
- ADB连接不稳定，尝试重新连接

### Q: 任务执行到某一步失败？

**A:** 检查：
1. 视频文件是否存在
2. 微信是否已登录
3. 手机分辨率是否匹配
4. 是否有弹窗遮挡

### Q: 网络ADB连接不上？

**A:**
```bash
# 检查手机和PC是否在同一WiFi
# 重启ADB服务
adb kill-server
adb start-server
# 重新连接
adb connect 手机IP:5555
```

## 📊 发布流程

系统自动执行以下步骤：

```
步骤1:  清空手机视频
步骤2:  推送视频到手机
步骤3:  等待视频加载
步骤4:  停止微信
步骤5:  等待关闭
步骤6:  启动微信
步骤7-11: 进入视频号
步骤12-17: 选择视频
步骤18-19: 切换输入法
步骤20-22: 输入描述
步骤23: 点击发表
步骤24: 完成
```
